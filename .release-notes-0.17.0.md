# DevDash v0.17.0

Build code goes further: Next.js template, GitHub auto-push, env hints from Settings.

## What's new

### Next.js 14 + Prisma template
- Full-stack App Router project — no separate backend folder, just one Next.js app.
- Postgres + Prisma 5 schema with `User`, `Role`, `Plan` enums.
- Cookie-based JWT auth (`/api/auth/register`, `/login`, `/me`, `/logout`, `/forgot-password`, `/reset-password`).
- Stripe Checkout, webhook, subscription state sync — webhook runs as a Node route with raw body handling.
- Admin routes + page (stats, user list with search, role/plan editing) wired into `/app/admin`.
- Frontend pages: Landing, Login, Register, ForgotPassword, ResetPassword, Dashboard, Pricing, Admin — all using Tailwind.
- `npm run make-admin <email> [password]` script to bootstrap an admin.
- Token replacement still works (`{{DISPLAY_NAME}}`, `{{PROJECT_SLUG}}`, `{{JWT_SECRET}}`).

### Create GitHub repo and push — one click
- Toggle "Create GitHub repo and push" in the Build code tab.
- Uses the GitHub token from Settings (set this in v0.13.0's Team tab field).
- Optional private-repo toggle. Default: private.
- Creates repo under your authenticated user, adds it as `origin`, renames branch to `main`, pushes first commit.
- New project entry in DevDash is auto-populated with the fresh `githubUrl`.
- Clear log lines: auth check, repo URL, branch rename, push output.

### Env hints from DevDash Settings
- New "Pre-fill .env hints from DevDash settings (when available)" toggle.
- Emits system log lines when your GitHub token is present.
- Does not write secrets into project files directly — designed to surface which keys you already have configured so you know what to supply.

### Build code tab polish
- "Open repo" button appears in the result bar after GitHub push succeeds.
- Privacy toggle hidden until GitHub push is enabled.
- GitHub push disabled when git init is unchecked (can't push without a repo).
- Result bar wraps cleanly when all four action buttons are shown (Open repo, Open folder, Open in VS Code, error).

## Under the hood

- `electron/scaffold.ts` now reads DevDash settings for the GitHub token and calls the GitHub API directly. No `gh` CLI dependency — same code path works on any Windows install.
- Templates folder bundled as `extraResources` (shipped in v0.16.0) now includes the Next.js variant. Installer size grew to ~83MB to accommodate.
- Scaffold result type extended with `githubUrl` so the auto-added DevDash project entry can link back to the repo.

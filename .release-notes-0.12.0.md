# DevDash v0.12.0

Stop manually copying env vars to Vercel/Render, and never lose your setup again.

## What's new

### Env sync to provider
- **↕ Sync Vercel/Render button** in the Env tab. DevDash compares your local `.env.production` (or `.env`, or `.env.example`) against the provider's environment variables and shows a per-key diff: only local, only remote, differ, or match.
- **Selective push** — tick the keys you want to sync (defaults to only-local + differ), click "Push to Vercel/Render", done. Values stay encrypted in transit; DevDash uses the provider's official APIs.
- **Masked values by default** with a per-row Reveal toggle, so you can compare without exposing secrets on your screen.
- **Vercel**: uses `POST /v10/projects/.../env?upsert=true` for new keys and `PATCH /v9/.../env/:id` for updates across production/preview/development.
- **Render**: uses `PUT /v1/services/.../env-vars` to merge selected keys into the existing set without clobbering anything.

### Quick backup (plain JSON)
- **Export backup** in Settings → Quick backup. Saves everything — projects, tokens, settings, and optionally the cache database (uptime history, deploy history, dep reports) — to a single JSON file via a native save dialog.
- **Restore backup** reads that JSON on another PC, snapshots the current config/cache as `.before-restore-<timestamp>.bak` first, and writes the new state.
- Separate from the existing **encrypted config backup**: use Quick backup for fast local migrations, encrypted backup for sharing or untrusted storage.

## Why

Env drift was the silent killer — "works on my machine" but Render still points at an old MongoDB URI. And a new laptop meant a full day of re-adding projects and pasting tokens. v0.12.0 closes both loops.

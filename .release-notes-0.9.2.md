# DevDash v0.9.2

Deploy actions upgrade. Redeploy any project without leaving the Deploys tab.

## New

**Redeploy button per row.** Every deployment card in the Deploys tab now has a Redeploy button. Triggers a fresh deploy on the same provider with one confirmation. Result toast shows at the top of the list so you know if it went through.

**Bulk redeploy.** A "Redeploy all" button in the Deploys header. One click triggers new deploys across every project that has a `deployId` configured (both Vercel and Render). Confirms the count before firing, then shows a summary toast with per-project failures listed inline.

**Result toast.** After any redeploy action, a dismissable banner appears showing how many succeeded and which failed with what error. Auto-clears on refresh.

## Why

The Redeploy action already existed on Projects cards, but you had to tab-hop from Deploys to Projects to find it. Now it's where you'd naturally look: right next to the deployment it'll replace.

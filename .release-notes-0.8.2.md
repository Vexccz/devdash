# DevDash v0.8.2

Patch release. Audit pass on stale references and one critical config bug.

## Fixed

**Critical: config persistence for new fields.** v0.8.0 added `automations` and `dbTargets` to the config schema, but `loadConfig()` was stripping unknown top-level fields on every read. Result: any automation or DB target you created would survive in-memory until the next app restart, then vanish. The merge logic now spreads parsed fields first, so additive schema changes are preserved.

**Run dev now resolves monorepo subfolders correctly.** v0.8.1 taught the framework detector to scan subfolders, but the standalone "Run dev (terminal)" entrypoint in main.ts was still using the old detector that only checked the root. It now uses the same detector as the in-app Logs tab, so a project root without a `dev` script will spawn from `frontend/`, `backend/`, etc.

## Polish

- Updated stale `User-Agent` strings (`DevDash/0.2.0` → `DevDash/0.8.1`) in screenshot capture and uptime checks.

## Install

Download `DevDash-Setup-0.8.2.exe` below. Existing installations upgrade in place; previously-saved automations/DB targets that were lost cannot be recovered, but new ones from this version onward will persist.

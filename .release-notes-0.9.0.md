# DevDash v0.9.0

Smart onboarding release. Goal: get your projects into the dashboard with one click instead of filling out forms.

## New

**Auto-detect on Add Project.** The Browse button now kicks off an inspection when you pick a folder. DevDash reads git remote URLs, detects the framework, and queries Vercel and Render (if you have tokens configured) to match the repo to an existing deploy. Name, GitHub URL, deploy provider, deploy ID, and live URL are filled automatically. An "Auto-detect" button next to Browse re-runs inspection on demand.

**Smart Import.** A new entry point in the Projects view header. Pick a parent folder (e.g. `C:\Users\zafra`) and DevDash scans every subdirectory for git repos or package files. Results show framework, matched deploy provider, and env status in a compact list. Select what you want and import in one click. Duplicate paths are skipped automatically.

**Vercel project matching by GitHub remote.** Uses the official `/v10/projects` endpoint, scans link metadata, and picks the first project whose GitHub owner/repo matches your local remote.

**Render service matching by GitHub remote.** Uses `/v1/services` and substring-matches on the repo URL field.

## How it works

- GitHub remote is read directly from `.git/config` (no git binary needed).
- SSH remotes (`git@github.com:...`) are normalized to HTTPS.
- Framework detection covers Next.js, Vite, Expo, React Native, Electron, Flutter, Python/FastAPI, and generic Node.
- Env hints show how many keys from `.env.example` are missing in `.env`.
- Deploy matching is best-effort: if there's no token or no match, the fields stay empty and you can fill them manually.

# DevDash v0.11.0

Smart dependency health: don't just blindly update, know what's safe and what'll break.

## What's new

- **npm audit integration** — security vulnerability counts (low / moderate / high / critical) per project, surfaced as an `audit` pill alongside the existing major/minor/patch counts.
- **Engine compatibility check** — DevDash reads `engines.node` from each project's `package.json` and compares against your installed Node. Mismatches show an "engine mismatch" badge with the expected vs. installed version on hover.
- **Safe update flow** — one click runs `npm update` (semver-respecting patches/minors only), `npm audit fix` (non-breaking), then verifies with `npm run build`. If the build fails, DevDash auto-rolls back `package.json` and `package-lock.json` and runs `npm install` to restore the previous tree. Major version bumps are always skipped.
- **Inline outdated package list** — click "Details" on any project to expand a per-package table (name, current, wanted, latest, semver-bump pill). Sorted with majors first.
- **Per-project Rescan** button so you don't have to scan everything when you just want to refresh one.
- **Result modal** showing applied updates, audit fixes count, build status, and rollback state.

## Why

"Main update je" was the dangerous shortcut. v0.11.0 splits the difference: patches and minors get applied automatically with build verification (because they almost never break), while majors stay manual because changelogs deserve to be read. Any breakage triggers a real rollback, not a "good luck" message.

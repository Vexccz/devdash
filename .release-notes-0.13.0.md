# DevDash v0.13.0

Manage GitHub collaborators without leaving DevDash.

## What's new

### Team tab
- **New "Team" tab** in every project's detail view, alongside Overview / Logs / Env / etc.
- **List collaborators** — avatars, usernames, and resolved roles (Admin, Maintain, Write, Triage, Read) pulled straight from the GitHub API.
- **Invite by username** with a permission dropdown (read through admin). DevDash sends the invitation; pending invites appear in a separate panel until the user accepts.
- **Remove collaborator** and **cancel pending invitations** inline, with confirmation prompts so you don't nuke someone by accident.
- **Click-through avatars** open the user's GitHub profile in your default browser.

### GitHub token in Settings
- New **GitHub token** field in Settings → API tokens. Needs classic `repo` scope, or a fine-grained token with Administration (read & write) + Metadata for the target repos.
- **Test** button reads `x-oauth-scopes` and the authenticated login so you know whether the token actually has what the Team tab needs before you try to invite someone.

## Why

Adding a teammate to a group project used to mean bouncing to github.com and digging through repo settings. Now it's three clicks from the same window where you're running the project. Pair this with v0.12.0's env sync and onboarding a new dev becomes: invite from DevDash, they clone, env sync from DevDash, `npm run dev`.

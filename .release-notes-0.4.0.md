DevDash 0.4.0 introduces a first-class chat panel backed by a local Ollama instance.

### Added
- **Chat tab** — new sidebar entry that connects to a local Ollama server (default `http://localhost:11434`, configurable in Settings via `ollamaBaseUrl`).
- **Model picker** — dropdown populated from `GET /api/tags` with a refresh button.
- **Streaming responses** — assistant output streams token by token through NDJSON parsing of `POST /api/chat`. A Stop button cancels the active stream.
- **Chat history** — conversations persist in SQLite (`chats`, `chat_messages` tables). The sidebar lists title, model, and relative updated time. Individual chats can be deleted.
- **Per-chat controls** — temperature slider (0.0–1.5) and system prompt field available directly in the composer row.
- **Settings surface** — `ollamaBaseUrl`, `ollamaDefaultModel`, `ollamaSystemPrompt`, and `ollamaTemperature` fields added to the application configuration with safe defaults.

### Changed
- Sidebar order is now Projects, Deploys, Uptime, Time, Deps, Chat, Settings.
- The command palette tab union has been expanded to include the new Chat tab.

### Known issues
- Assistant responses currently render as monospaced plain text. Markdown rendering and code-block syntax highlighting are planned for a follow-up.
- When Ollama is not running, the Chat tab surfaces an unreachable hint but does not automatically retry. Use the refresh button to reconnect.
- The remainder of the v0.4.0 roadmap (keyboard shortcuts overlay, encrypted configuration export and import, light mode, scheduled automations, pull request status, diff viewer, database health, Render service metrics, Vercel analytics, project detail breadcrumb, and the Electron major-version upgrade) remains deferred.

### Install
Download `DevDash-Setup-0.4.0.exe` below. Please uninstall the previous version first for a clean install. Existing configuration and project data are preserved across upgrades.

# DevDash v0.10.0

First-deploy release. Create new Vercel projects and Render services without leaving DevDash.

## New

**One-click first deploy.** Projects without a deploy provider configured now show a "🚀 Deploy" button. Clicking opens a modal that:

1. Pre-fills framework, build command, and output directory based on detected project type (Vite, Next.js, Flutter, Node).
2. Imports environment variables from your local `.env` automatically.
3. Lets you pick Vercel or Render with provider-specific options.
4. Calls the official APIs (`/v10/projects` for Vercel, `/v1/services` for Render) to create the project, attach env vars, and trigger the first build.
5. Updates the local DevDash config with the new deploy ID and live URL so subsequent operations (Redeploy, status polling, metrics) work immediately.

**Vercel options:** framework slug, build command, output directory, root directory (monorepo support), branch, environment variables.

**Render options:** service type (web service or static site), region (Singapore, Oregon, Frankfurt, Ohio), plan (free, starter, standard) for web services, branch, environment variables.

**Auto-import .env values.** When opening the deploy modal, DevDash reads the project's local `.env` file and pre-fills it as encrypted environment variables on the new deployment. Edit, remove, or add before submitting.

## Workflow

Before v0.10.0: deploy first via Vercel/Render UI → copy IDs back into DevDash → use Redeploy.

With v0.10.0: click Deploy in DevDash → fill defaults → done. The project links automatically.

## Install

Download `DevDash-Setup-0.10.0.exe` below. Existing projects are unaffected; the new button only appears on projects without a deploy provider configured.

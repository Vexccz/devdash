# DevDash v0.15.0

Build Capacitor APKs without the Java-version dance.

## What's new

### Mobile tab
- **New "Mobile" tab** in every project detail. Detects Capacitor automatically by reading `package.json` and `capacitor.config.{ts,js,json}`, surfacing app id, app name, web dir, Capacitor version, and whether the `android/` folder is wired up.
- **Java compatibility check** — DevDash compares your installed Java major version (read from `java -version`) against what your Capacitor version expects (Cap 6 → JDK 17, Cap 7+ → JDK 21). Mismatches show a yellow warning card with a hint instead of a cryptic Gradle stack trace later.
- **One-click APK build** with three configurable steps and a flavor toggle:
  - Web build (`npm run build`)
  - `npx cap sync android`
  - `gradlew assembleDebug` / `assembleRelease`
  - Each step is optional via checkboxes. Useful when you've already built and just want to re-assemble.
- **Live build log** streamed back over IPC. stdout/stderr coloring, system messages in indigo, scrolling tail, clearable.
- **Copy to Downloads** option saves the APK as `<ProjectName>-<flavor>-<timestamp>.apk` next to the original Gradle output, with a "Show in folder" shortcut on success.
- **Build duration** + path shown on completion. Failures bubble up the actual exit code and, when relevant, the Java mismatch hint.

## Why

Cap 6 expects JDK 17, Cap 8 expects JDK 21. Half the APK build failures aren't even code problems, they're "I forgot which Java is on PATH". DevDash now tells you that upfront, runs the whole pipeline (`build → sync → assemble`) for you, and drops the APK in Downloads. No more terminal hopping.

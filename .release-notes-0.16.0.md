# DevDash v0.16.0

Scaffold a production-ready SaaS in one click, right inside DevDash.

## What's new

### Build code (sidebar entry)
- **New "Build code" tab** in the sidebar.
- **Pick a template** — currently two:
  - `React + Express + MongoDB` — Vite + Tailwind frontend, Express + Mongoose backend, JWT auth, Stripe Checkout + portal + webhook, admin panel, password reset, welcome / payment-success emails (Resend with console fallback).
  - `React + Express + Postgres (Prisma)` — same feature set running on Postgres + Prisma, with `prisma:migrate` script.
- **Configure** project name, display name, parent folder (native picker), Stripe toggle, npm install, git init.
- **Live build log** streams stdout/stderr/system messages over IPC while DevDash runs `copyDir` → `npm install` (frontend + backend) → `git init && commit`.
- **Auto-register** the new project — when scaffolding succeeds, DevDash adds it to the project list with the chosen path. Open it in VS Code or run `npm run dev` straight from Projects.
- **Token replacement** — `{{PROJECT_NAME}}`, `{{PROJECT_SLUG}}`, `{{DISPLAY_NAME}}`, `{{JWT_SECRET}}` (random 32-byte hex), and `{{STRIPE_KEY_HINT}}` are replaced across every file. JWT secret is fresh on every scaffold.

### Ships with both backends
- bcrypt password hashing
- JWT auth (`/register`, `/login`, `/me`, `/forgot-password`, `/reset-password`)
- Stripe Checkout, billing portal, webhook handler with payment-success email
- Admin routes — user list with search, role/plan editing, stats endpoint (totals, conversion rate, last 24h)
- Email service via Resend, falls back to logging when `RESEND_API_KEY` is empty so nothing breaks before keys are set
- `make-admin <email> [password]` script to bootstrap your first admin

### Frontend pages included
- Landing, Login, Register, Pricing, Dashboard, ForgotPassword, ResetPassword, Admin
- Auth context with token refresh, protected routes, dashboard-to-admin link for admin role

## Why

Half a day of "set up auth, set up Stripe, set up admin, set up email" each new SaaS project was the same boilerplate over and over. v0.16.0 turns that into one click and a fresh `npm run dev`.

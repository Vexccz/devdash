DevDash 0.3.0 ships three high-impact features on top of the critical fixes previewed in 0.3.0-batch1.

### Added
- **Project tags** — projects accept free-form tags in the Add/Edit modal. A filter bar above the project grid allows multi-select filtering. Tags persist in configuration.
- **Quick commit** — a new Commit action on each project card opens a modal showing the working-tree status, a commit-message textarea, and optional stage-all / push-after-commit checkboxes. Executes via `simple-git` for staging and commit, and the hardened git wrapper for push.
- **Manual redeploy** — projects configured with a Vercel project ID or Render service ID gain a Redeploy action that triggers a fresh production deployment via the provider API.

### Fixed (carried from 0.3.0-batch1)
- Sentry error-budget integration now resolves the organization slug dynamically instead of using the previously hardcoded value.
- Git progress output no longer generates PowerShell error objects on Windows.
- `npm audit` count reduced via non-breaking dependency bumps; the lone critical advisory is eliminated.

### Known issues
- Fifteen security advisories that require breaking dependency upgrades (Electron, Vite, @electron/rebuild) remain deferred to a later release.
- The larger v0.3.0 scope (Ollama chat panel, keyboard shortcuts overlay, configuration export and import, light mode, scheduled automations, pull request status, diff viewer, database health, Render metrics, Vercel analytics, project detail breadcrumb) is planned for v0.4.0.

### Install
Download `DevDash-Setup-0.3.0.exe` below. Please uninstall the previous version first for a clean install. Existing configuration and project data are preserved across upgrades.

# DevDash v0.8.1

Patch release. Two bug fixes addressing common monorepo and connection setup pain points.

## Fixed

**Run dev now works for monorepo projects.** The framework detector previously gave up if no `dev` script existed at the project root. It now scans common subdirectories (`frontend`, `backend`, `web`, `client`, `app`, `apps/web`, `packages/web`, `server`, `api`) and runs from the first one that resolves. The detected framework label includes the subdirectory in parentheses for clarity.

## New

**Auto-detect database targets from env files.** A new "Auto-detect from env" button on the DB Health tab scans every project's `.env`, `.env.local`, `.env.production`, etc., looking for `MONGODB_URI`, `DATABASE_URL`, `POSTGRES_URL`, and similar keys. Detected URLs are classified by protocol (`mongodb://` / `postgres://`) and added as targets automatically. Duplicates are skipped. Subdirectories `backend`, `server`, `api`, `frontend` are also scanned.

# DevDash v0.8.0

Monitoring and automation release. Four new features covering scheduled tasks, database health, and deployment analytics.

## New

**Automations** — cron-scheduled tasks per project. Auto-pull or auto-deploy on any valid cron schedule. Six presets included (30m, hourly, daily, weekly). History of recent runs with success/error status. Each run logs to SQLite for later inspection. Jobs survive app restarts and are stored in config.

**Database Health** — ping MongoDB and Postgres connections directly from the dashboard. Connection strings stored locally (never sent elsewhere). Shows latency, server version, and up/down status per target. Multiple targets per project supported. Password values are masked in the UI.

**Render Metrics** — CPU and memory graphs for Render services. Configurable window (1h / 6h / 24h / 3d). Uses the official Render metrics API; resolution fixed at 5-minute intervals. Rendered as inline sparklines with latest-value annotations.

**Vercel Analytics** — traffic summary for Vercel projects with Web Analytics enabled. Shows total visitors, pageviews, daily trend, and top pages. Gracefully degrades when analytics isn't enabled on the project side with clear guidance.

## Polish

- Breadcrumb navigation in project detail modal (Projects / Project / Tab).
- Command palette now includes entries for the three new tabs.
- Sidebar reorganized with icons for Automations, DB Health, and Metrics.

## Technical

- New IPC handlers: 12 (automation: 7, dbhealth: 5, metrics/analytics: 2)
- New dependencies: `mongodb@6.10.0`, `pg@8.13.1`
- No breaking changes to existing config schema; new fields are additive (`automations`, `dbTargets`)
